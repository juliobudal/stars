    - Plan 07-03 will replace this file with the real cache logic

  <action>
    Create `app/views/pwa/service-worker.js` with this exact content:

    ```js
    // PWA Service Worker — see Plan 07-03 for full cache strategy implementation.
    // This stub exists so the /service-worker route resolves with 200 instead of 404.
    self.addEventListener("install",  () => self.skipWaiting());
    self.addEventListener("activate", () => self.clients.claim());
    self.addEventListener("fetch",    () => {});
    ```

    The empty `fetch` handler is critical: Chrome's installability check requires a fetch event listener exist (even if it does nothing). Without this, Lighthouse PWA audit fails the "service worker registered" check.
  </action>
